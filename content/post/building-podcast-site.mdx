# Building a Podcast Site with Git-based CMS

## Content Structure
```typescript
content/
  episodes/
    episode-1.mdx
    episode-2.mdx
  about.mdx
  contact.mdx
```

## Custom Components
```typescript
// AudioPlayer.tsx
<AudioPlayer 
  url={episode.audioUrl}
  title={episode.title}
  duration={episode.duration}
/>

// EpisodeCard.tsx
<EpisodeCard
  title={episode.title}
  date={episode.date}
  description={episode.description}
  thumbnail={episode.thumbnail}
/>
```

## Handling Dynamic Content

### Comments System Options

1. **Third-party Services**
```typescript
// Using Giscus (GitHub Discussions)
<Giscus
  repo="your-username/your-repo"
  repoId="your-repo-id"
  category="Comments"
  categoryId="your-category-id"
/>
```

2. **Simple Backend**
```typescript
// Lightweight server with SQLite
interface Comment {
  id: string;
  episodeId: string;
  content: string;
  author: string;
  createdAt: Date;
}
```

## User Experience

### Content Management
- Episode content in MDX
- Show notes with markdown
- Transcripts with custom formatting
- Static pages for about/contact

### Interactive Features
- Episode player
- Episode navigation
- Search functionality
- Category filtering

<Callout type="tip">
  Keep static content in MDX files and use third-party services or a simple backend for dynamic features like comments and analytics.
</Callout>
